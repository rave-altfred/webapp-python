📚 Metric Explanations

📹 RTSP Reader Metrics: • Service Status: Overall health status of the RTSP Reader service
• Uptime: Time since last service restart
• Streams: Total configured vs actively running RTSP streams
• Publisher: Message publisher connection status and performance
• Circuit Breaker: Protection mechanism status (Closed = normal, Open = error protection)
• Queue Stats: Message publishing queue size and error counts
🎯 Computer Vision Service Metrics: • Service Status: Overall health of the CV processing service (healthy/degraded/unhealthy)
• Valkey Connection: Connection status to Valkey message queue (connected/degraded/disconnected)
• Consumer Status: Whether the message consumer is actively processing frames (running/stopped)
• Messages Processed: Total number of frames processed by the CV pipeline
• Memory Usage: Current RAM usage percentage (warning >60%, critical >80%)
• CPU Usage: Current processor utilization by the service
• Uptime: Time since last service restart
🗄️ Valkey Database Metrics: • 🚨 PEL Messages: Messages in Pending Entry List - delivered to consumers but not ACKed yet. This is the "hidden queue" that causes processing delays!
• Total Queue: All pending messages (PEL + undelivered). PEL messages are the real problem in stream processing.
• Oldest Pending Age: How long the oldest PEL message has been waiting (indicates consumer processing delays)
• PEL Backlog Time: Estimated time to clear all PEL messages at current processing rate (16s/message)
• Hit Ratio: % of successful key lookups vs total lookups (higher is better for performance)
• Memory: Current RAM usage by the Valkey database
• Clients: Number of active client connections
• Process Rate: Commands processed per second (in detailed view)
🐘 PostgreSQL Metrics: • Database Size: Total disk space used including all tables, indexes, and data
• Cache Hit Ratio: % of data found in memory vs read from disk (>95% excellent, 90-95% good, <90% needs attention)
• Active Connections: Database connections currently executing queries
• Idle in Transaction: Connections holding open transactions (should be low)
• Rollback Ratio: % of transactions that failed (lower is better, <5% is good)
• Blocks Read/Hit: Data blocks from disk (slower) vs memory (faster)
• Tuples Returned: Total number of rows returned by all queries
📋 Table Health: • Live Tuples: Current number of active/visible rows in each table
• Dead Tuples: Rows marked for deletion but not yet cleaned up by VACUUM
• Health Status: Good (<10% dead), Fair (<20% dead), Needs Vacuum (>20% dead)
• Bloat %: Ratio of dead tuples to live tuples (indicates need for maintenance)
• Operations: Total INSERT/UPDATE/DELETE operations since last statistics reset
